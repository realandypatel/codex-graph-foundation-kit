# SAFE_CLAIMS.md

## Safe public claims

You can safely say:

- This is a GitHub-ready starter kit for graph-first Codex workflows.
- It helps teams configure MCP, prompts, repo memory, and operating rules for cleaner agent workflows.
- It is designed to reduce context waste by encouraging graph-first discovery and targeted file reads.
- It works best on larger, multi-file, dependency-heavy tasks.
- Actual gains vary by repository and task.

## Claims to avoid

Do not say:
- guaranteed 90% token reduction
- identical to the upstream graph engine
- better on every task
- universally faster on trivial edits

## Safe benchmark wording

You may say:

Public benchmarks from the upstream `code-review-graph` project report an average 8.2x token reduction across tested repositories, with larger gains on bigger multi-file tasks and weaker gains on trivial single-file changes.

## Important distinction

This kit is:
- a starter foundation
- an operational system
- a documentation + config + prompt layer

This kit is not:
- the graph engine itself
- the benchmark suite itself
- a guarantee of the same benchmark results in every repo
