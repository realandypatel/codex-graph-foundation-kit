## LinkedIn post

I packaged a reusable foundation kit for graph-first Codex workflows.

The goal is simple: stop letting coding agents re-read huge parts of the repo every time. Instead, use graph-first discovery, repo memory, and cleaner session structure so work stays targeted and consistent across new chats.

This starter kit includes:
- Codex / MCP setup guidance
- prompts for setup, opening, task blocks, and session closure
- repo memory templates
- graph-ignore defaults
- verification scripts
- safe public wording for teams sharing the workflow

Important note:
This is a foundation kit, not the graph engine itself. It is designed to work with upstream graph tooling such as `code-review-graph`.
